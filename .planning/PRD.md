# Product Requirements Document (PRD)

## Product Name
CrowdLife H5

## Product Vision
Build a mobile-first interactive social world where users discover moving characters and build persistent relationships through realistic conversations.

## Target Users
- Users seeking emotional companionship or curiosity-driven social discovery
- Users who enjoy narrative interaction and role-based dialogue

## Core User Problems
- Existing chatbots feel generic and lack social context.
- Conversations rarely carry meaningful continuity across sessions.
- No strong “discovery” layer before conversation begins.

## MVP Goal
Validate that users enjoy tapping moving characters and returning for ongoing conversations with remembered context.

## Core Scenarios
1. User opens mobile H5 and observes a living crowd.
2. User taps a moving character.
3. Character approaches and starts conversation.
4. Character references prior interaction when user returns later.

## Functional Scope

### In MVP
- Two-way moving crowd scene
- Tap-to-approach-to-chat loop
- Distinct role personas
- Persistent user-character memory
- Relationship state progression basics
- Safety guardrails and failure fallback

### Not in MVP
- Full voice interaction
- User-generated character creation
- Multiplayer user-to-user social mechanics

## Success Metrics
- Character click-through rate from crowd scene
- Conversation completion rate
- 7-day return rate to same character
- Subjective realism score from user feedback
