## 当前产品目标

把 PepiTalk 从“用户点开角色聊天的页面”升级为：

**一个有活人感、有关系推进、有共同城市世界观、正常模式与探索模式互相打通的沉浸式互动产品。**

## 核心优化方向

### A. 正常模式：从“点人聊天”升级为“进入在线社会”

目标：用户不是旁观者，而是被这个世界注意到的人。

### B. 探索模式：从“AI 生成故事”升级为“成熟互动叙事”

目标：用户感觉自己真的在推进事件，并且选择会带来后果。

### C. 双模式打通：关系线 + 世界线 + 事件线联动

目标：平时聊天埋线索，故事推进反过来影响角色关系与表达。

## 一、体验总目标清单

### 1. 活人感

- [ ] 角色不再只被动等待用户点开，开始具备“主动出现”的能力
- [ ] 角色会根据关系阶段、时间段、情绪状态变化主动程度
- [ ] 用户能感受到“今天这个角色状态不一样”
- [ ] 用户能感受到“这个角色记得我，而且对我有印象”

### 2. 关系感

- [ ] 正常模式里的关系推进不只靠聊天轮数，而是靠“状态变化”
- [ ] 角色对用户形成不同印象，例如：会听人说话、嘴硬、深夜活跃、好奇心强
- [ ] 角色会根据用户过往互动调整语气、边界、主动性
- [ ] 用户能体验到“关系事件”，例如第一次主动找你、第一次向你袒露秘密、第一次明显吃醋/担心/回避

### 3. 世界感

- [ ] 核心角色不再彼此孤立，而是共享同一座城市世界
- [ ] 固定的地点、事件、传闻会在不同角色口中反复出现
- [ ] 不同角色对同一件事有不同证词和态度
- [ ] 用户能感受到“我在同一张社会网络里移动”

### 4. 作品感

- [ ] 探索模式每条故事都有清晰结构：开场钩子、调查展开、中段反转、情绪结局
- [ ] 选择不再只是换个说法，而是真的改变人物态度、线索顺序、结局倾向
- [ ] NPC 在故事中具备自己的行动线，不只是等待用户点选
- [ ] 每条故事结尾都应有情绪余味，而非仅解释设定

## 三、正常模式升级 TODO

### P0：先把“社会感”做出来

- [ ] 设计“主动来找你”机制的产品规则
  - 建议包括：触发时机、频率控制、角色差异、关系门槛、时间段限制
- [ ] 设计“路过式互动”规则
  - 例如：打招呼、一句短问、轻吐槽、提一个小问题，而不是每次都进入完整深聊
- [ ] 设计“角色今日状态”规则
  - 例如：忙、想说话、低落、心情好、欲言又止、对某件事在意
- [ ] 设计“用户社会印象”规则
  - 例如：角色如何形成对用户的判断，以及判断会如何影响互动

### P0：把关系推进从“轮数”升级为“状态”

- [ ] 定义通用关系阶段
  - 建议：陌生 → 认脸熟 → 熟络 → 特别信任 / 若即若离 / 共同秘密
- [ ] 定义每个阶段的说话边界与主动性差异
- [ ] 定义“关系事件”类型库
  - 例如：第一次主动找你、第一次叫昵称、第一次提起旧事、第一次请求建议
- [ ] 为 6 个核心角色分别补充关系推进节点

### P1：让角色之间通过用户发生连接

- [ ] 设计“弱连接台词”规则
  - 例如：A 提到 B，B 提到同一地点，或 A/B 对同一传闻表达不同态度
- [ ] 设计“城市流言”规则
  - 让角色偶尔提到同一城市里最近发生的小事
- [ ] 设计“跨角色回响”规则
  - 例如：用户在一个故事里经历过的事，会改变其他角色的说法

### P1：让用户不是空气，而是这个世界的一员

- [ ] 设计用户在世界中的“存在痕迹”表达方式
- [ ] 设计“用户影响到别人关系/情绪/走向”的反馈方式
- [ ] 设计“不是所有关系都线性增长”的真实感规则

验收标准：用户在正常模式下，即使不主动频繁点人，也能感受到有人会注意到自己、关系会自然推进、不同角色属于同一个社会。

## 四、探索模式升级 TODO

### P0：先把故事结构专业化

- [ ] 为每条主故事定义固定 4 幕结构
  - 第一幕：异常钩子
  - 第二幕：调查展开
  - 第三幕：中段反转
  - 第四幕：情绪结局
- [ ] 为每幕定义明确功能，避免每幕都只是“叙事 + 选项”重复
- [ ] 规定每条故事至少有一次“原来不是我以为那样”的反转
- [ ] 规定结尾必须有情绪余味类型
  - 例如：释然、怅然、轻惊悚、未尽、和解、代价感

### P0：把选择做出真正后果

- [ ] 定义选择后果类型库
  - 人物态度变化
  - 线索先后顺序变化
  - 场景进入顺序变化
  - 结局倾向变化
  - 错过某条支线信息
- [ ] 规定每次关键选择至少改变一个后续变量
- [ ] 设计“延迟反馈”机制
  - 不是选完立刻说教，而是两幕之后让用户发现前面的选择在生效

### P1：让 NPC 成为真正的人，而不是功能点

- [ ] 为探索模式 NPC 增加独立动机
- [ ] 为探索模式 NPC 增加隐瞒、试探、主动出现、态度变化机制
- [ ] 设计“矛盾证词”规则，增强调查感和专业感
- [ ] 设计“NPC 不在场时也会行动”的表达方式

### P1：增强影视化与现场感

- [ ] 制定探索模式的场景描写标准
  - 具体、可见、可听、可触，不堆空泛形容词
- [ ] 制定“镜头推进感”规则
  - 从远景到细节，从环境到人物，从静态到事件发生
- [ ] 制定“微叙事”规则
  - 路人、小物件、推送、旧海报、声音残留等细节增强真实感

验收标准：用户在探索模式中能够明显感受到节奏变化、选择后果、角色动机和结局余味，而不是只是在看 AI 续写文本。
